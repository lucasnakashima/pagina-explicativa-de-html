Client x Server

Clients

(Voltar)

     No desenvolvimento moderno de software, os Clients compreendem o hardware e o software que durante o seu funcionamento acessam recursos disponibilizados por um Server. Esse software pode ser tanto uma aplicação instalada em um computador quanto uma acessada pela Internet com o auxílio de um navegador. Exemplos de Client incluem...

No caso de aplicações instaladas:

  • Loja de aplicativos (Play Store)
  • Aplicativos de mensagem instantânea (Whatsapp, Facebook)
  • E-mail (Outlook, Gmail)

No caso de aplicações Web:

  • Plataformas de vídeo (Youtube)
  • Sites de banco
  • Redes sociais (Twitter, Facebook)

Nesse sentido, é importante mencionar que um mesmo software pode ser estar disponível tanto como aplicação instalada quanto como aplicação web.

Servers

(Voltar)

     Os Servers, nesse mesmo contexto, compreendem o hardware e o software que atendem as requisições por recursos realizadas pelos Clients. Como esse trabalho geralmente é pesado, o hardware deles costuma ser robusto e, assim como o software no qual é executado, ele é voltado a esse tipo de tarefa. Assim, possuem sistemas operacionais e computadores próprios diferentes daqueles dos Clients. Tipos de Servers incluem...

  • Arquivos
  • Firewall
  • Streaming
  • E-mail
  • Web

Linguagens de Programação

(Voltar)

     As Linguagens de Programação são línguas formais concebidas para a descrição das instruções de programas de computador, expressando o seu algoritmo. Elas são concebidas para atender necessidades e problemas que surgiram ao longo do desenvolvimento da tecnologia e de suas aplicações, podendo ser voltadas para determinada área. Dessa forma, existem línguas voltadas as aplicações do Client, enquanto outras são direcionadas ao Server.

No lado do Server, elas incluem...

  • Javascript(Node.js)
  • Java
  • C#
  • Ruby
  • PHP

HTML

Inspetor de Elementos

(Voltar)

     O Inspetor de Elementos é uma ferramenta disponibilizada pelo navegador para verificar o código HTML e CSS associado a determinado elemento da página visualizada. Apesar de não compor o HTML, ele é um meio interessante de estudar o código de um site e aprender com ele.

Estrutura de um arquivo HTML

(Voltar)

Um arquivo HTML simples pode ser visto a seguir:

<html>
	<head>
		<title>Minha página</title>
	</head>
	<body>
		<h1>Título da página</h1>
		<h2>Subtítulo da página</h2>
		<p>Parágrafo</p>
		<ol>
			<li>Elemento da lista</li>
			<li>Elemento da lista</li>
			<li>Elemento da lista</li>
		</ol>
		<ul>
			<li>Elemento da lista</li>
			<li>Elemento da lista</li>
			<li>Elemento da lista</li>
		</ul>
	</body>
</html>

Nesse exemplo:

  • <html> e </html> delimitam o arquivo HTML;
  • <head> e </head> delimitam os elementos da página que aparecem ao usuário antes de o site carregar;
  • <title> e </title> delimitam o nome da aba da página;
  • <body> e </body> delimitam os elementos apresentados depois de o site carregar;
  • <h1> e </h1> delimitam um cabeçário;
  • <h2> e </h2> delimitam um cabeçário menor;
  • <p> e </p> delimitam um parágrafo;
  • <ol> e </ol> delimitam uma lista ordenada, numerada;
  • <ul> e </ul> delimitam uma lista não ordenada;
  • <li> e </li> delimitam um elemento de uma lista

Tags e suas funções

(Voltar)

     As Tags no HTML podem tanto corresponder a uma estrutura (cabeçário, parágrafo, lista) como a um meio de estilizar essas estruturas. Assim, além das tags já apresentadas, existem:

  • <mark> e </mark> que marcam o texto;
  • <small> e </small> que deixam o texto menor;
  • <i> e </i> que deixam o texto em itálico;
  • <u> e </u> que deixam o texto sublinhado;
  • <abbr> e </abbr> que sinalizam uma abreviatura (HTML);
  • <del> e </del> que sinalizam que um texto foi deletado;
  • <font> e </font> que altera as características do texto;
  • <sup> e </sup> que deixam o texto sobrescrito;
  • <sub> e </sub> que deixam o texto subscrito;
  • <blockquote> e </blockquote> que deixam o texto em bloco, como uma citação:
"Penso, logo existo." (Descartes)

<h3> e </h3>

<h4> e </h4>

<h5> e </h5>
<h6> e </h6>
